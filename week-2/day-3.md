# Week 2 Day 3 — Layered Remote State Architecture

## Learning Objective

Students will understand **Layered Remote State Architecture** and apply it in a real enterprise Terraform platform.

---

## Topics

- Layer separation
- Cross-layer dependency
- terraform_remote_state

---

## Real Hands-on Tasks

- Connect networking output to compute layer
- Read storage output from another layer

---

## Lab Instructions

1. Open the week-2 lab folder.
2. Review the weekly `README.md`.
3. Create or update the required Terraform files.
4. Run Terraform commands safely.
5. Validate the result.
6. Commit the completed work.

---

## Recommended Commands

```bash
terraform fmt -recursive
terraform init
terraform validate
terraform plan
```

For apply sessions:

```bash
terraform apply
```

---

## Student Deliverables

- Working Terraform code
- Clean folder structure
- Updated README notes
- Successful Terraform plan output
- Git commit for the completed day

---

## Notes for Instructor

Focus on real-world explanation.

Explain why this pattern is used in enterprise infrastructure teams.
